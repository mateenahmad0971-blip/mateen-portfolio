Site Report

A reflective account of my web development journey this term.

Introduction

This report documents my experience developing a portfolio website as part of the CSY1063 Web Development module at the University of Northampton. Over the course of this term, I have learned to design, code and deploy a multi-page responsive website using HTML5 and CSS3. This has been both a challenging and deeply rewarding process, and I want to capture the highs and lows honestly.

Getting Started — Early Weeks

When I first started, I had very little experience with HTML and almost none with CSS. Writing my first index.html felt daunting — I kept second-guessing every tag and wondering if I was doing things correctly. The biggest early hurdle was understanding the difference between block and inline elements, and why my divs kept behaving unexpectedly.

One of my first wins was getting the navigation bar to look the way I imagined it. I used a semantic nav element with an unordered list inside, and styled it with Flexbox. Seeing it snap into place was a genuinely satisfying moment that motivated me to keep going.

Design Decisions

I deliberately chose a dark theme for the portfolio, using a near-black background (#0d0f14) with a bright cyan-green accent (#39ff8f). I wanted the site to feel technical and modern — something that reflected a developer's identity rather than a generic template aesthetic.

For typography, I selected Syne for headings (a geometric display font that feels bold and distinctive) and DM Mono for code snippets and labels, giving the site a programming feel. I found both fonts through Google Fonts after experimenting with several combinations. I was inspired by minimalist developer portfolios I found on Dribbble and Awwwards, particularly their use of generous negative space and strong type hierarchies.

The binary-number background pattern was a deliberate choice to reinforce the computer science theme without being distracting — it's subtle and low-opacity, acting as texture rather than noise.

Technical Development

Learning CSS Grid was one of the most important technical steps I took. I initially tried to build layouts with Flexbox alone but quickly realised Grid offered far more control for two-dimensional layouts. The projects page uses a masonry-style grid, which is different from the single-column layout on the home and report pages, demonstrating grid versatility.

Making the site responsive was the most time-consuming part. I created separate CSS stylesheets for each page's unique layout, alongside a shared base.css for the navigation, footer, and common utility styles. Media queries handle screens below 768px. The most complex challenge was the hamburger menu — I used a button element and toggled a CSS class with a small JavaScript event listener. Getting the animation smooth and the menu overlay working correctly took several iterations.

I used CSS custom properties (variables) throughout base.css for colours and spacing, which made maintaining consistency across five pages much easier. Changing the accent colour everywhere was a single-line edit.

Ups and Downs

There were moments of real frustration. At one point my whole layout broke after I accidentally applied a conflicting style rule, and it took me over an hour to track down the issue using browser developer tools. That experience taught me to use DevTools far more actively for debugging.

On the positive side, completing the contact form (using a mailto: action) and seeing it open my email client for the first time was a small but real milestone. The code validation stage was also satisfying — running all files through the W3C validator and watching them pass gave a sense of completeness.

Reflections and Improvements

Looking back, I would have started using GitHub commits much earlier. In the beginning I only saved files locally and didn't commit until I had built several pages. I now understand why regular, meaningful commits matter — they act as a safety net and document the story of a project's development.

If I were to improve the site further, I would add JavaScript-driven animations for page transitions, implement a dark/light mode toggle, and replace the mailto form with a proper server-side solution using PHP or a form service like Formspree. I would also explore adding a blog section to document ongoing learning.

Overall, this module has given me a solid foundation in front-end web development. I leave it with skills I can genuinely apply and build on throughout my degree and beyond.
